# Project Planning & Execution

This directory serves as the source of truth for the Notary Agent Application's development lifecycle. As a development team, we use this space to track completed milestones, manage the current sprint, and outline future architectural and feature goals.

## Directory Structure
- `01-completed-work.md`: A historical log of all major architectural decisions, UI implementations, and completed sprints.
- `02-current-sprint.md`: The active tasks being worked on by the team right now.
- `03-future-sprints.md`: The roadmap and backlog of upcoming features.

## Workflow Rules
1. **Branching**: We use a strict `main` -> `feat/` -> `development/` workflow. 
2. **Updates**: Before merging a feature branch into `main`, the developer must update these planning documents to reflect the newly completed work and adjust the upcoming sprints accordingly.
